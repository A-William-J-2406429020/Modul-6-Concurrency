Di Milestone 1 ini, saya mempelajari cara menginisialisasi server di Rust menggunakan std::net::TcpListener yang dibind ke lokal. Melalui perulangan incoming(), server dapat mendeteksi setiap upaya koneksi TCP yang masuk dari klien atau browser. Saya melihat bahwa setiap kali alamat lokal diakses, terminal mencetak pesan konfirmasi koneksi, namun seringkali muncul beberapa kali karena perilaku browser yang melakukan permintaan berulang.  Sistem ini masih bersifat single-threaded, sehingga jika ada satu proses yang memakan waktu lama, koneksi lain akan tertunda. Hal ini memberikan pemahaman dasar bagaimana aliran data atau stream ditangani sebelum nantinya dioptimalkan dengan multi-threading.

Di Milestone 2 ini, saya mempelajari cara mengirim file HTML ke klien menggunakan fs::read_to_string untuk membaca isi file hello.html. Saya memahami bahwa respons HTTP harus memiliki format spesifik yang mencakup status line, header, dan badan pesan. Header Content-Length sangat diperlukan agar browser mengetahui ukuran data yang akan diterima sebelum merendernya. Penggunaan makro format! memudahkan penggabungan berbagai variabel status dan konten menjadi satu string respons yang utuh. Terakhir, data tersebut dikirimkan kembali melalui stream TCP dalam bentuk bytes menggunakan fungsi write_all.

Di Milestone 3 ini, saya belajar memvalidasi request agar server bisa memberikan respons berbeda tergantung URL yang diakses oleh pengguna. Saya menggunakan logika if/else untuk membedakan antara permintaan ke halaman utama (/) dan permintaan ke halaman yang tidak tersedia. Jika pengguna mengakses URL yang salah, server akan secara otomatis mengirimkan status 404 NOT FOUND beserta isi dari file 404.html. Kode awal saya memiliki banyak duplikasi pada bagian pembacaan file dan pengiriman stream, sehingga saya melakukan refactoring untuk menyederhanakannya. Refactoring ini sangat penting dilakukan agar kode lebih mudah dipelihara serta menghindari kesalahan logika akibat pengulangan baris yang sama. Dengan memisahkan penentuan status dan nama file ke dalam variabel, struktur fungsi handle_connection kini menjadi jauh lebih bersih dan efisien.
